# Collection Orchestrator 最终仲裁与主题报告模板

## 固定输入

- 集合目录：`{COLLECTION_DIR}`
- 集合 manifest：`{COLLECTION_MANIFEST}`
- 逐书卡片：`{PER_BOOK_CARDS}`
- Agent C 输出：`{AGENT_C_OUTPUT}`
- Agent D 输出：`{AGENT_D_OUTPUT}`
- claims ledger：`{CLAIMS_LEDGER}`
- 必要原文回查记录：`{BOOK_MD_LOOKUPS}`
- 用户额外要求：`{USER_REQUIREMENTS}`
- 输出目标：`{OUTPUT_TARGET}`

不得读取任何 `source/` 原始书籍文件。需要原文核验时，只能回查对应书籍的 `conversion/book.md`。

## 任务

你是主线程 Collection Orchestrator。请基于已验证的单书产物、Agent C 横向综合、Agent D 横向审查、claims ledger 和必要原文回查记录，仲裁并生成唯一正式主题共读报告：`{OUTPUT_TARGET}`。

## 仲裁原则

- 已验证单书深读结果是基础，不得用主题分类替代单书章节覆盖。
- 跨书判断必须可追溯到 `synthesis/claims_ledger.md`。
- 若 Agent C 与 Agent D 冲突，以单书最终总结、结构材料和 `conversion/book.md` 回查为准。
- 无法确认的内容标为残余风险，不写成确定结论。
- 不得把多位作者合并成一个集体观点，除非 claims ledger 明确支持。
- 不得为了形成漂亮框架抹平冲突。
- 不得默认生成批判性评价；只有用户明确要求时才单独增加批判性评价章节。

## 正式报告结构

使用 `SKILL.md` 的输出语言优先级：用户明确指定语言优先，其次使用用户请求语言，请求语言不明时使用英文。请按以下结构写作：

1. `主题问题界定`
   - 说明本次共读回答的问题和边界。

2. `这组书分别站在哪里`
   - 分别定位每本书，不合并作者立场。

3. `共识区`
   - 只写 claims ledger 支持的共识，并标明证据等级。

4. `分歧区`
   - 保留解释权重、方法、叙事视角和概念使用上的分歧。

5. `互补区`
   - 说明各书如何在尺度、时代、方法、案例或叙事角度上互补。

6. `不能被强行综合的张力`
   - 写出必须保留的不可化约冲突。

7. `关键概念矩阵`
   - 简明呈现概念在各书中的不同含义和使用方式。

8. `论证图、时间线或叙事图`
   - 按这组书的体裁选择最合适的结构图。

9. `推荐阅读路径`
   - 给出阅读顺序和理由。

10. `可追溯 claims ledger 摘要`
    - 摘要核心 claims、证据等级、信心和风险。

11. `残余风险与未解决问题`
    - 列出缺失材料、未回查原文、高风险归因和未解决分歧。

如果用户明确要求批判性评价，再增加独立章节：`批判性评价`。该章节可讨论证据质量、方法局限、偏见、遗漏和当代相关性。

## 交付与顶层目录约束

- 最终只保留一个正式主题报告：`{OUTPUT_TARGET}`。
- 不要在集合目录顶层生成或保留 Agent C、Agent D、仲裁记录、长篇质检报告或其他中间材料。
- C/D 输出、仲裁记录和验证记录应保留在 `synthesis/` 或 `synthesis/review/`。
- 交付前必须运行集合验证器，并修复所有失败项。

## 会话质量说明

最终会话回复只给浓缩质量说明：

- 每本书均已完成单书流程和验证。
- Agent C/D 是否启动；若未启动，说明具体原因和自审路线。
- 核心仲裁结果。
- claims ledger 的处理结果。
- 残余风险。
